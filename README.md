# CWatch 项目说明

本项目是基于 RT-Thread、SiFli SDK 和 LVGL 的智能手表应用工程，当前仓库主体以手表固件工程为主，另外本地还配有微信小程序工程用于联调。

- 手表固件工程
- BLE 数据同步链路
- 经典蓝牙音频连接模板

## 当前功能概览

### 手表侧

- 传感器采集与共享数据中心
- LVGL 本地界面显示
- BLE 广播与 GATT 数据通道
- 经典蓝牙音频连接模板
- 接收手机侧天气、时间等同步数据

### 小程序侧

- 自动初始化蓝牙
- 自动查找并连接名为 `CWatch` 的 BLE 设备
- BLE 通道建立后自动同步：
  - 当前时间
  - 今天天气
  - 7 天天气
  - 24 小时天气
- 保留手动补发入口，便于自动同步失败时重试

## 主启动流程

`src/main.c` 当前按如下顺序初始化：

1. `DataHub_Init()`
2. `SensorTask_Start()`
3. `lvgl_start()`
4. `bt_classic_app_init()`
5. `ble_app_init()`

其中：

- `bt_classic_app_init()` 用于注册经典蓝牙相关事件，当前作为音频类连接模板保留
- `ble_app_init()` 负责 BLE 使能、服务初始化和广播启动

## 模块职责

### `src/app/`

- `data_hub.*`：共享运行时数据模型（`system_data`）与互斥锁（`data_mutex`）
- `sensor_task.*`：传感器采集线程（IMU、环境、心率血氧、磁力计、RTC）
- `ui_update.*`：周期刷新 UI，并按页面动态启停采集项
- `app_ble.*`：BLE 初始化、广播、连接事件与服务启动
- `ble_rx.*`：BLE 收包、时间/天气等手机数据解析
- `app_bt_classic.*`：经典蓝牙音频连接模板，供后续手机音频播放能力扩展
- `compass.*`：基于运动与磁力数据计算航向角

### `src/drivers/`

- `BME280/`：温度、气压、湿度
- `LSM6DS3TR/`：加速度计与陀螺仪
- `MAX30102/`：心率与血氧
- `QMC5883/`：磁力计
- `RTC/`：实时时钟
- `sensor_i2c/`：传感器共用 I2C 抽象层

### `src/ui/`

- `ui.c/ui.h`、`ui_start.c`：UI 初始化与 LVGL 线程启动
- `screens/`：生成页面与自定义页面（`ui_Screen1...7`、天气页等）
- `images/`、`fonts/`：LVGL 图片/字体 C 数组资源
- `components/`：UI 组件钩子
- `honeycomb.*`：蜂窝启动器布局逻辑

### `CWatch_vxxcx/`

- `pages/index/`：连接与同步首页
- `pages/logs/`：小程序默认日志页
- `utils/util.js`：基础工具函数

## 蓝牙方案说明

当前工程采用 BLE 与经典蓝牙并存的方式：

- BLE：
  - 用于小程序数据通信
  - 负责时间、天气等业务数据同步
  - 当前广播已调整为更标准的可发现广播格式，便于手机扫描
- 经典蓝牙：
  - 当前提供音频类连接模板
  - 主要用于后续扩展手机音乐在手表播放等场景
  - 不替代 BLE 数据通道

也就是说：

- 手机系统蓝牙连接成功，不等于小程序已经拿到 BLE 数据通道
- 小程序的数据同步仍然依赖 BLE
- 目前通过“小程序自动补 BLE 连接”的方式实现用户无感连接

## 运行时数据流

1. 传感器线程采集数据并写入 `system_data`
2. UI 定时器根据当前页面启用所需采集组
3. UI 读取数据快照并刷新控件
4. BLE 接收手机侧数据（如天气、时间）并更新 `system_data` 与 RTC
5. 小程序在 BLE 通道就绪后自动发起同步流程

## 微信小程序说明

`CWatch_vxxcx/` 为微信小程序工程，当前主要用于手表数据同步。

### 连接流程

1. 打开首页
2. 自动初始化蓝牙
3. 优先从当前已知设备中查找 `CWatch`
4. 若未找到，则开始扫描
5. 发现目标设备后自动建立 BLE 连接
6. 获取服务与特征值
7. 通道就绪后自动同步数据

### 自动同步内容

- 时间同步
- 今天天气同步
- 7 天天气同步
- 24 小时天气同步

### 手动同步入口

首页保留了手动补发按钮，仅作为自动同步失败后的兜底入口，不改变原有协议和功能内容。

## Git 提交说明

当前仓库默认纳入版本管理的是固件主体源码与工程配置。

微信小程序目录 `CWatch_vxxcx/` 当前默认不纳入仓库，用于本地联调与页面开发。

## 说明与建议

- `project/build_CWatch_hcpu/` 属于构建产物，应视为生成目录
- 若后续需要单独开源或提交小程序，可再单独调整 `.gitignore`
- 当前经典蓝牙部分是连接模板，后续若要实现真正的音频播放，还需要继续补音频输出链路
